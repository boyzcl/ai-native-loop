# Pattern Intake Template

## Purpose

不是每个 field note 都值得升级成 pattern。

这个模板的作用，是把原始任务记录压成 pattern 前，先过一层统一筛选。

## Intake Questions

把候选样本压缩回答以下问题：

- `scene`
  这类任务通常发生在什么场景？
- `trigger`
  哪些信号说明该触发这个模式？
- `intervention`
  默认介入等级是什么，为什么？
- `artifacts`
  最该优先产出哪些工件？
- `common_failure`
  最容易错在哪？
- `transfer_rule`
  它为什么能迁移到其他任务？

## Promotion Rule

只有在下面三项都能答清时，才值得升级成 pattern：

1. 触发信号清楚
2. 工件选择稳定
3. 可迁移规律明确

如果只能讲一个案例故事，还不能讲清迁移规律，就先保留为 field note。
