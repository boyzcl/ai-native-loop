# Self Bootstrap Pattern

## Scene

一个系统、项目、流程或 Skill 需要评估自己、识别缺口、修补自己，并把结果推进到公开发布或下一轮状态。

## Trigger

出现以下信号时触发：

- 用户要求“评估当前完成度并继续补齐”
- 当前任务不是做一个单点产出，而是治理整个工作链路
- 评估、改进、验证、发布之间是强耦合关系

## Intervention

默认 `medium`，必要时上调到 `strong`。

推荐动作：

1. 先出 `Diagnosis Card`，确认断点在设计层、交付层还是发布层。
2. 再出 `Task Packet`，把“评估”和“补齐”串成一条推进链。
3. 发现缺口后立即转为 `Feedback Attribution Card`。
4. 把反馈改写成一轮 `Re-input Packet`，直接进入修补动作。

## Artifacts

优先产出：

- 完成度评估
- 缺口清单
- 执行计划
- 发布工件
- 下一轮迭代入口

## Common Failure

最常见失败：

- 把任务退化成“打一分”
- 识别出缺口但不转成动作
- 只有自评，没有对外验证

## Transfer Rule

这个模式不只适用于 Skill，也适用于：

- 项目复盘与升级
- 产品发布前治理
- 个人工作流系统优化

核心迁移规律：

> 把“评估”当成下一轮执行的输入，而不是最后的结论。
