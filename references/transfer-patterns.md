# Transfer Patterns

## 核心原则

这个 skill 不按领域切成多个互不相干的小技能，而是用同一套协议迁移到不同知识工作中。

共同协议始终不变：

- 重写输入
- 组织执行
- 读取反馈
- 改写再输入

## 编码与构建

- 输入：把需求、代码状态、报错和验收标准重写成任务包。
- 执行：拆出探索、修改、验证和回收。
- 反馈：读取测试结果、运行日志、review 发现。
- 再输入：将失败模式压缩为下一轮修改指令。

## 研究与分析

- 输入：把研究问题、范围、证据边界和输出形式讲清。
- 执行：区分检索、比较、归纳和判断。
- 反馈：读取证据缺口、冲突来源和可信度不足。
- 再输入：收缩问题或补关键证据。

## 写作与表达

- 输入：把目标读者、要传达的核心、已有素材与限制讲清。
- 执行：区分结构搭建、素材选择、文风控制和事实核对。
- 反馈：读取结构问题、表达问题、说服问题和风格偏差。
- 再输入：改写提纲、段落职责或信息密度要求。

## 产品思考

- 输入：把用户问题、场景、约束和验证目标重写出来。
- 执行：区分问题定义、方案展开、验证假设和优先级判断。
- 反馈：读取价值错位、验证不足、路径过宽等信号。
- 再输入：收缩假设、补验证或重写目标用户。

## 决策整理

- 输入：明确决策对象、可选项、标准和风险。
- 执行：区分信息收集、选项生成、权衡比较和拍板。
- 反馈：读取标准不稳、信息不全、风险未显化。
- 再输入：先补标准，再补材料，不急着给结论。

## Agent 协作

- 输入：把总目标拆成可交接子任务。
- 执行：定义每个 agent 的责任边界、依赖物和回收格式。
- 反馈：读取交接失败、重复劳动和上下游不对齐。
- 再输入：改写子任务合同与汇总协议。

## 多轮推进

- 输入：不是重复讲背景，而是明确“上一轮产出了什么，本轮要推进什么”。
- 执行：每轮只推进一个主要矛盾。
- 反馈：每轮至少沉淀一个稳定可复用资产。
- 再输入：减少重复叙述，增加结构化递进。
