# Intervention Matrix

## 设计目的

动态介入是这个 skill 的核心。不要默认上强结构，也不要在循环已坏掉时只给轻提醒。

## 判断维度

快速扫六个维度：

- 目标清晰度
- 上下文完整度
- 风险与可逆性
- 运动部件数量
- 反馈可见性
- 协作复杂度

## 轻介入

适合：

- 用户知道自己在做什么，只是表达松散。
- 当前轮次以推进速度为主。
- 失败成本低，可边做边修。

动作集：

- 把原始请求压缩成一段可执行输入。
- 补 1 到 2 个关键字段。
- 给出下一检查点与更好问法。

不要做：

- 输出大篇方法论。
- 引入复杂模板。
- 代替用户重定义目标。

## 中介入

适合：

- 已经有材料和动作，但质量波动明显。
- 当前问题不在“有没有做”，而在“做得不成循环”。
- 一个任务牵动多个输出或多个阶段。

动作集：

- 重组任务包。
- 明确人机或多 agent 分工。
- 定义成功标准、检查点和回收物。
- 给出下一轮再输入结构。

不要做：

- 一上来替用户拍板所有方案。
- 把中等复杂度任务升级成重治理工程。

## 强介入

适合：

- 任务高风险、跨轮、跨角色或长期漂移。
- 已经连续失败，但每一轮都没有真正改写输入或分工。
- 用户要的是结果，但当前更需要的是先把系统搭稳。

动作集：

- 暂停直接解题。
- 重建目标、约束、角色、材料边界与验证路径。
- 拆出本轮必须收口的最小问题。
- 明确哪些结论现在不能替用户做。
- 输出一份可持续运行的闭环协议。

不要做：

- 用强介入包装空泛概念。
- 忘记回落到下一步可执行动作。

## 升级信号

出现以下任一情况，提升一档：

- 同类错误重复至少两轮。
- 反馈内容只是在重复抱怨，没有归因。
- 用户提供的大量材料无法汇成一个当前任务包。
- 有多个 agent 或角色，却没有明确交接物。
- 输出方向反复跑偏。

## 降级信号

出现以下情况，可以降一档：

- 目标、交付物、评估标准已经稳定。
- 用户与 ai 的分工已形成固定节奏。
- 反馈可以被快速吸收到下一轮，不再需要重治理。

## 经验法则

- 轻介入解决“表达不够好”。
- 中介入解决“循环不够稳”。
- 强介入解决“工作系统本身失真或断裂”。
