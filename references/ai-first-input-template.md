# AI-first Input Template

## 用途

当用户给的是一堆材料、背景、想法或历史对话，而不是一个当前轮次可执行的输入时，用这份模板把信息压缩成 AI-first 输入。

## 模板

```md
## what-we-are-trying-to-move
- 这一轮真正要推进的对象：

## why-now
- 为什么这一轮值得现在推进：

## what-ai-needs-to-know-now
- 当前必须知道的事实：
- 当前必须知道的约束：
- 当前不能忽略的背景：

## what-can-wait
- 暂时不进入本轮的材料：

## preferred-role-of-ai
- 这轮更希望 ai 扮演什么角色：重写、比较、草拟、分析、执行规划、验证辅助

## expected-output-shape
- 希望返回什么形状：任务包、比较表、提纲、决策框架、修改方案、下一轮输入包

## evaluation-lens
- 这轮主要看什么：正确性、完整性、速度、清晰度、可执行性、可迁移性
```

## 使用原则

- 先压缩当前轮次，再谈全局背景。
- 不要把所有材料都扔给 ai，要明确哪些是本轮必读，哪些可以暂缓。
- 明确 ai 在这一轮扮演的角色，能显著减少输出漂移。
