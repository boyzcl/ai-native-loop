# Task Rewrite Template

在输入阶段，可把原始请求重写为以下模板：

```md
## objective
- 这一轮真正要推进的目标：

## current-state
- 已有内容：
- 当前卡点：

## artifacts
- 必读材料：
- 可选补充：

## constraints
- 时间/格式/质量要求：
- 不可做事项：

## success-signal
- 什么结果算这一轮有效：

## next-checkpoint
- 做到哪里必须停下来检查：
```

## 使用原则

- 优先补关键字段，不补废话。
- 如果某字段未知，直接标未知，不要假装完整。
- 需要轻介入时，只输出缺口最大的字段即可。
