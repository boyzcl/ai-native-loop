# Failure Modes

## Why This File Exists

协议型 Skill 最容易退化的方式，不是能力不够，而是总在同一类误判上重复犯错。

这份文件专门沉淀高频失败模式。每条都包含：

- 典型症状
- 真正根因
- 纠偏动作

## 1. 过早求答案

症状：

- 用户材料很多，但直接要求最终方案
- AI 一上来开始写结论

根因：

- 输入阶段没有收敛出当前轮目标

纠偏：

- 先输出 `Diagnosis Card`
- 把任务压成 `Task Packet`
- 先定义这轮成功信号，再决定是否进入产出

## 2. 把结构问题误判成模型能力问题

症状：

- 用户说“AI 不行”“输出质量差”
- 但每轮输入都在换说法，没有补结构

根因：

- 真正坏的是上下文组织方式，而不是模型本身

纠偏：

- 优先检查 `artifacts`、`constraints`、`success_signal` 是否缺失
- 不先换模型，先补任务包

## 3. 只有材料，没有本轮目标

症状：

- 一次性塞进来大量文档、链接、代码、笔记
- 但说不清这一轮到底要推进什么

根因：

- `objective` 缺失，导致所有材料都在竞争注意力

纠偏：

- 强制收敛为一个本轮推进目标
- 只保留和该目标直接相关的材料进入当前 `Task Packet`

## 4. 只有目标，没有验收标准

症状：

- 目标听起来明确，但做完了也不知道算不算好

根因：

- `success_signal` 缺失

纠偏：

- 要求至少写出一个可判断的成功信号
- 没有成功信号时，不进入细节生产

## 5. 反馈停留在情绪层

症状：

- “不太对”“一般”“还是不行”
- 没有说明具体哪类问题

根因：

- 没有把反馈转成归因结构

纠偏：

- 输出 `Feedback Attribution Card`
- 强制把反馈归入 failure class

## 6. 重复失败但协议不变

症状：

- 同类任务做了两三轮
- 每次都只是在重写答案，不改输入和分工

根因：

- 再输入环节缺失

纠偏：

- 停止继续生成新答案
- 先产出 `Re-input Packet`
- 明确下一轮到底改什么

## 7. 轻介入不够，却仍然只做润色

症状：

- 任务持续跑偏
- 但输出仍只是“更好问法”或“更清晰表达”

根因：

- 介入等级判断过低

纠偏：

- 检查六个风险维度
- 达到 3 项以上异常时，升级到 `medium`

## 8. 强介入过度，拖慢推进

症状：

- 任务并不复杂，却引入大段协议和多张模板
- 用户真正需要的是快速推进

根因：

- 介入等级判断过高

纠偏：

- 回到最小充分原则
- 只保留 `Diagnosis Card` + `Task Packet`

## 9. 多 Agent 协作没有交接物

症状：

- 多个 Agent 同时工作
- 但主 Agent 最后拿不到稳定可回收结果

根因：

- 没有定义交接物和回收格式

纠偏：

- 在开始前指定 handoff artifact
- 每个 Agent 至少返回：做了什么、发现了什么、下一步建议什么

## 10. 把 AI 当黑箱代理替用户决策

症状：

- 用户把目标判断和价值取舍一起外包给 AI
- AI 开始替用户做不可逆选择

根因：

- 人机分工失配

纠偏：

- 把决策权显式收回给用户
- AI 只负责展开选项、压缩信息、分析权衡

## 11. 输出很多，但没有回收物

症状：

- 每轮看起来都产出了不少内容
- 下一轮却几乎从头开始

根因：

- 当前输出没有被压成下一轮可用工件

纠偏：

- 每轮结束至少保留一份 `Re-input Packet`
- 中介入以上必须明确回收物

## 12. 案例很多，但经验不复用

症状：

- 仓库里有案例
- 但新任务来时，不能快速知道能复用哪种方法

根因：

- 没有模式层

纠偏：

- 把案例压成 pattern 文件
- 每篇 pattern 必须明确可迁移规律

## 13. 把研究结构直接当成决策结构

症状：

- 输出讲了很多机制、成分、事实或背景
- 但用户仍然无法做出选择
- AI 容易用“强推荐”掩盖真正的决策边界

根因：

- 研究层没有被改写成决策层
- 任务对象仍然是“解释”，而不是“帮助判断”

纠偏：

- 先重写决策对象、选项边界与成功信号
- 用 `Task Packet` 明确决策分层，而不是继续扩写研究综述
- 用 `Re-input Packet` 把研究框架折叠成选项、权衡和边界

## Fast Triage Rule

出现问题时，优先按这个顺序排查：

1. 有没有本轮目标
2. 有没有成功信号
3. 介入等级是否判断错误
4. 反馈有没有进入归因结构
5. 下一轮输入有没有被改写

如果前两项都缺，先不要继续解题。
