# Case 02: mRNA Research To Report

## Case Summary

这个案例记录的是：一项围绕 `Rosie` 犬类个性化 mRNA 癌症疫苗事件的复杂研究任务，如何从一个高模糊度的调查目标，推进为一条可执行、可校验、可成稿的研究与写作链路。

它不是“让 AI 写篇文章”的案例，而是一个更典型的外部知识工作案例：

- 起点是一个值得调查的复杂事件
- 中间需要处理研究问题、公开信息、可信度差异和技术流程
- 终点不是一段答案，而是一组有分工的交付物

因此，这个案例最适合被定义为：

> `ai-native-loop` 在研究分析与写作表达场景中的迁移案例：把复杂调查任务组织成输入、执行、反馈、再输入的稳定闭环。

## Why This Case Matters

这个案例重要，是因为它补上了 `Case 01` 之外最关键的一块证据：

- 它不是 Skill 自己评估自己
- 它不是元任务
- 它是一个真实的外部知识工作场景

而且它同时跨了两个常见场景：

- 研究分析
- 长文写作

这正好能验证 `ai-native-loop` 的主张并不依赖编码任务，而是可以迁移到更广义的知识工作中。

## Original Working Goal

原始目标可以概括为：

- 深度调查 Paul Conyngham 使用 ChatGPT 与 AlphaFold 为患癌犬 Rosie 设计个性化 mRNA 癌症疫苗的完整事件
- 还原该事件的技术流水线
- 形成既能供调查阅读、又能供技术复盘的成稿材料

如果直接把它理解成“写一篇文章”，这个任务会很快失控。因为它天然同时包含：

- 新闻调查
- 事实校验
- 技术背景检索
- 生物医学术语翻译
- 技术流程重建
- 最终成稿整合

这类任务最容易出现的问题，不是不会写，而是：

- 目标范围不断膨胀
- 事实与推断混在一起
- 中间材料无法回收
- 最终文章难以建立可信结构

## The Existing Artifact Chain

这个案例的强度很高，因为它已经天然形成了一条完整工件链：

- 原始执行协议：[执行指令.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/执行指令.md)
- 中间调查产物：[findings.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/findings.md)
- 技术流水线手册：[pipeline.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/pipeline.md)
- 最终整合成稿：[report.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/report.md)

这意味着它不是只有“最后一篇文章”，而是保留了从任务定义到中间层再到终稿的全过程证据。这非常适合被 `ai-native-loop` 复盘。

## How The Skill Would Reframe The Task

`ai-native-loop` 不会把这个任务简单理解为：

- 搜资料
- 写报道
- 总结一个 AI 案例

它更合适的重写方式是：

1. 先把目标拆成“事件调查”和“技术流程重建”两条并行主线。
2. 再把每条主线都拆成可回收的中间工件。
3. 用可信度标注和信息缺口控制研究质量。
4. 最后把两条主线重新合并成面向读者的整合报告。

换句话说，这个 Skill 在这里最重要的作用，不是“帮忙多搜一点”，而是：

- 先定义信息结构
- 再推进研究执行
- 再读取中间反馈
- 最后重写成稿输入

## Intervention Judgment

这个案例最适合的介入强度是 `medium`，并且在部分阶段会短暂上调到 `medium-high`。

原因是：

- 目标本身清晰，但执行链路复杂
- 材料来源多，可信度不一
- 中间必须保留可回收工件
- 最终输出不止一种，而是报告加技术手册

它不需要 `strong` 级别的从零重构，因为主题和交付方向并不含混；但也绝不只是 `light`，因为如果没有中间协议，任务会迅速退化成一堆无序检索和一篇不稳的长文。

## How The Loop Showed Up In Practice

### 1. 输入重写

任务没有被写成一句“帮我调查一下这个事”，而是被重写成了一份高约束的执行协议。

这一点体现在 [执行指令.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/执行指令.md)：

- 先定义角色
- 再定义调查目标
- 再规定分阶段执行
- 再规定每一阶段都要输出中间结论
- 最后明确要落成 `findings.md`、`pipeline.md` 和 `report.md`

这就是很典型的 `ai-native-loop` 式起手：

- 不直接追求终稿
- 先把工作系统搭出来

### 2. 执行组织

任务执行时，并不是只围绕“收集更多信息”，而是围绕“让每条信息有位置”展开：

- 时间线信息进入调查主线
- 技术细节进入技术主线
- 文献背景进入支撑层
- 已确认、合理推断、信息缺口被显式区分

这一点在 [findings.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/findings.md) 和 [pipeline.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/pipeline.md) 里都很明显。

这说明这个过程的执行质量，不只是“信息多不多”，而是“信息有没有被组织成后续可继续推进的结构”。

### 3. 反馈读取

这个案例里，反馈并不主要来自“有人告诉你这篇写得不好”，而来自研究过程本身暴露出来的信号：

- 哪些内容是第一手来源
- 哪些内容只能做高可信推断
- 哪些关键细节仍然公开缺失
- 哪些技术节点适合写进手册，哪些只能保留为限制说明

这类反馈被显式折叠进了 [findings.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/findings.md) 的“信息可信度地图”和“局限性与争议”结构里。

这正符合 `ai-native-loop` 的一个核心判断：

> 不要只读取输出，要读取反馈；不要只知道写到了哪里，要知道哪里还不能写得太满。

### 4. 再输入改写

这个案例最能体现 Skill 效果的地方，是中间产物没有停留在“备忘录”，而是直接变成了下一轮输入。

具体表现是：

- 调查结果沉淀为 `findings.md`
- 技术重建沉淀为 `pipeline.md`
- 这两个文件再被折叠为 [report.md](/Users/boyzcl/Documents/A/存量文章撰写/mRNA/report.md)

也就是说，这个过程天然完成了：

- 一轮研究
- 一轮结构化沉淀
- 一轮成稿整合

这就是 `输入 -> 执行 -> 反馈 -> 再输入` 在研究与写作场景中的直接体现。

## Concrete Skill Effects

如果从 `ai-native-loop` 的角度看，这个案例至少验证了六个具体效果：

### 1. 它让复杂调查任务先变成任务包

不是直接搜，而是先规定角色、范围、阶段、输出和可信度规则。

### 2. 它让中间过程可回收

很多研究写作任务的问题，不是没结果，而是中间过程不可复用。这个案例里，中间层被保留下来了。

### 3. 它把“文章”拆成了多层交付物

最终不是一篇稿，而是：

- 调查报告
- 技术流水线手册
- 整合型长文

### 4. 它让反馈进入结构，而不是停留在感觉

信息可信度、推断边界和公开缺口都有明确位置，这会直接提高最终成稿的稳健性。

### 5. 它证明同一协议可以迁移到研究和写作

这个案例不依赖编程任务，也不依赖项目管理语境，但依然非常适合 `ai-native-loop`。

### 6. 它让下一轮工作更容易继续

后续如果要继续做：

- 补文献
- 做图表
- 改写成公众号长文
- 压缩成演讲提纲

都已经有结构化基础，而不是要从头再来。

## Why This Is A Strong Public Case

这个案例适合公开放进仓库，主要因为它同时满足四个条件：

- 有真实任务
- 有完整中间链路
- 有清晰前后结构变化
- 有跨场景迁移价值

相比很多“AI 帮我写了一篇东西”的案例，它更能说明：

- Skill 改变的是工作方式
- 不只是输出速度

## Limits Of This Case

这个案例也有明确边界：

- 它更强地证明了研究与写作场景，不等于已经覆盖所有知识工作
- 它展示了高质量协议型任务定义，但还不是一段完整的人机逐轮对话记录
- 其中部分技术细节本身仍依赖公开资料的可得性，因此案例应强调“结构价值”，而不是包装成医学结论验证

因此，它非常适合作为 `Case 02`，但后面仍建议继续补：

- 产品推进案例
- 决策整理案例
- 多 Agent 协作案例

## Final Assessment

这个案例可以作为 `ai-native-loop` 的首个外部主题案例。

推荐给它的定位是：

> `Case 02`：验证 `ai-native-loop` 如何把一个复杂研究任务，推进为可调查、可归因、可成稿、可继续迭代的知识工作闭环。

它和 [case-01-self-bootstrap.md](/Users/boyzcl/Documents/AI%20native/ai-native-loop/docs/cases/case-01-self-bootstrap.md) 形成了一个很好的组合：
