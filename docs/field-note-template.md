# Field Note Template

## Purpose

这不是长篇复盘模板，而是经验沉淀的最小 intake 表。

只记录后续能复用的字段，不记录所有过程细节。

在 `medium` 及以上介入里，默认先保留一个更短的 `Loop Recovery Block`。只有当样本值得进入经验系统时，再扩写成完整 field note。

## Loop Recovery Block

```md
Loop Recovery Block
- scene:
- initial_block:
- artifacts_produced:
- what_worked:
- remaining_risk:
- next_input:
```

使用原则：

- 这是默认回收块，不是完整 field note
- 先求低摩擦留下痕迹，再决定是否继续提升
- 对 `medium` 及以上介入，默认把它放在本轮输出末尾

## Promotion Checklist

决定是否从 recovery block 升级成 field note 前，先检查：

- 这轮是否暴露了重复失败或新失败模式？
- 这轮是否形成了可迁移的做法？
- 这轮是否改变了之后的触发、分工或评估方式？
- 这轮是否值得进入 `pattern` / `failure_mode` / `benchmark`？

至少命中两项，再优先升级为完整 field note。

## Template

```md
# Field Note: <short title>

## Scene

- 这是什么类型的任务？

## Objective

- 这轮真正要推进什么？

## Initial Block

- 一开始卡在哪里？

## Intervention Level

- `light` / `medium` / `strong`
- 为什么是这个等级？

## Core Artifacts Produced

- 实际产出了哪些工件？

## What Changed

- 相比原始请求，最关键的结构变化是什么？

## What Worked

- 哪个动作或工件最有效？

## What Failed Or Remained Risky

- 哪些问题还没解决？
- 暴露了什么失败模式？

## Re-input

- 下一轮输入应该如何改写？

## Promotion Hint

- 更像 `pattern` / `failure_mode` / `benchmark` / `raw_note`？
- 为什么？
```

## Use Rule

优先在以下情况记录 field note：

- 一个任务明显暴露了新失败模式
- 一个做法看起来可以迁移
- 一个场景值得进入 benchmark
- 这轮输出虽然成功，但只有记录下来下次才不会重做

如果只是想保留最小痕迹，而不是立刻升级经验资产，先写 `Loop Recovery Block` 即可。
